# Career Project

Небольшой **REST API на Node.js**: можно добавлять криптовалюты, а цены подтягиваются с Binance и сохраняются в базу.

**Что умеет сервер:**

- создавать, смотреть, менять и удалять валюты;
- раз в 10 секунд обновлять цены с Binance;
- отдавать сохранённые цены по тикеру (например `BTC`);
- проверять токен в заголовке запроса;
- показывать документацию в браузере на `/docs`.

**Нужно:** Node.js версии 22 или новее.

---

## Как запустить

1. Установить зависимости:

```bash
npm install
```

2. Создать в корне проекта файл `.env`:

```env
AUTH_TOKEN=мой-секретный-токен
PORT=3000
```

`AUTH_TOKEN` — обязательный. Без него авторизация не заработает.

3. Запустить:

```bash
npm start
```

4. Открыть в браузере: `http://localhost:3000/docs` — там можно посмотреть все методы API.

База данных создаётся сама в папке `data/` при первом запуске.

---

## Как пользоваться API

В каждый запрос нужно добавить заголовок (подставь свой токен из `.env`):

```
Authorization: Bearer мой-секретный-токен
```

### Список методов

| Что сделать               | Метод  | Адрес                 |
| ------------------------- | ------ | --------------------- |
| Проверить, что сервер жив | GET    | `/status`             |
| Все валюты                | GET    | `/currency`           |
| Одна валюта               | GET    | `/currency/1`         |
| Добавить валюту           | POST   | `/currency`           |
| Изменить валюту           | PUT    | `/currency/1`         |
| Удалить валюту            | DELETE | `/currency/1`         |
| Цены по тикеру            | GET    | `/price?currency=BTC` |

### Пример: добавить Bitcoin

Тело запроса (JSON):

```json
{
  "name": "Bitcoin",
  "ticker": "BTC"
}
```

Через Postman или curl — главное не забыть заголовок `Authorization`.

## Из чего состоит проект

Код разложен по папкам — у каждой своя задача:

```
index.js          ← здесь всё собирается и запускается сервер
config.js         ← настройки из .env

src/
  routes/         ← принимают HTTP-запросы (что пришло с клиента)
  services/       ← бизнес-логика (правила, проверки)
  repository/     ← работа с базой (SQL-запросы)
  database/       ← подключение к SQLite и создание таблиц
  middleware/     ← auth (токен) и обработка ошибок
  errors/         ← свои типы ошибок (404, 400 и т.д.)
  utils/          ← логгер и вспомогательные функции
```

**Как идёт обычный запрос:**

```
Запрос → auth (проверка токена) → route → service → repository → база
                                                      ↓
                                              errorHandler (если ошибка)
```

**Как обновляются цены:**

```
Каждые 10 сек → scheduler → priceService → Binance → сохранение в базу
```

Так проще поддерживать код: в роутах нет SQL, в сервисах нет HTTP.

## Настройки (.env)

| Переменная        | Зачем                                 |
| ----------------- | ------------------------------------- |
| `AUTH_TOKEN`      | Секрет для входа в API (обязательно)  |
| `PORT`            | Порт сервера (если не указать — 3000) |
| `BINANCE_API_URL` | Адрес Binance (можно не трогать)      |
| `NODE_ENV`        | `development` или `production`        |
